# CoDroid ROS 2 双臂控制接口文档

## 1. 范围与数据约定

本接口只控制左右机械臂各 7 个关节，不控制头部、灵巧手或夹爪。

```text
LeftArm:  J_arm_l_01 ... J_arm_l_07
RightArm: J_arm_r_01 ... J_arm_r_07
```

关节角为 `rad`，速度为 `rad/s`，力矩为 `N·m`，笛卡尔位置为 `m`。

## 2. 控制链路

```text
目标 PoseStamped
  → 控制器 IK / 本地 IK
  → 目标 7 轴关节角
  → 从最新实际关节角开始三次插值
  → 100 Hz /codroid/arm_command
  → UDP LeftArm.P / RightArm.P (:9001)
  → 机器人
  → UDP AP/AV/AT (:9002)
  → /joint_states
```

Pose 接口只对最终 Pose 做一次 IK，随后执行关节空间插值，末端路径不保证为直线。

## 3. 构建与启动

```bash
cd /home/ym/workspace/Human02
colcon build --base-paths codroid_description \
  --packages-select codroid_description --symlink-install
source install/setup.bash

ros2 launch codroid_description realtime_arms.launch.py \
  robot_ip:=192.168.2.16 ik_solver:=controller_then_local
```

启动不会自动上使能、获取控制权或开启左右臂运动控制。

### Launch 参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `robot_ip` | `192.168.2.16` | 控制器 IP |
| `command_port` | `9001` | UDP 指令端口 |
| `feedback_port` | `9002` | UDP 状态接收端口 |
| `auto_connect` | `true` | 启动时发送 `StartFlag:true` |
| `frequency_hz` | `100.0` | 插值下发频率 |
| `default_max_velocity` | `0.25` | 自动时长使用的关节速度上限，rad/s |
| `ik_solver` | `controller` | `controller`、`local` 或 `controller_then_local` |
| `rviz` | `true` | 是否启动 RViz |
| `base_height` | `1.5` | `world` 到 `base_link` 的场景高度，m |

轨迹节点还支持参数：`minimum_duration=1.0`、`state_timeout=0.5`、
`hold_final_position=true`、`ws_port=9000`、`ik_timeout=3.0`。

## 4. Topic

### `/joint_states`

- 类型：`sensor_msgs/msg/JointState`
- 方向：桥接节点发布
- 映射：控制器 `AP → position`、`AV → velocity`、`AT → effort`

用于 RViz 显示、当前关节状态和插值起点。Pose/轨迹命令要求存在不超过
`state_timeout` 的完整反馈。

### `/codroid/status`

- 类型：`std_msgs/msg/String`
- 方向：桥接节点发布
- 内容：控制器原始状态 JSON

```bash
ros2 topic echo /codroid/status
```

### `/codroid/arm_command`

- 类型：`sensor_msgs/msg/JointState`
- 方向：桥接节点订阅
- 作用：底层实时关节位置入口，直接转换为 UDP `P`

带名称时必须包含完整左臂、完整右臂或完整双臂。不带名称时必须恰好包含 14 个位置，
顺序为左 7 轴、右 7 轴。

```bash
ros2 topic pub --once /codroid/arm_command sensor_msgs/msg/JointState \
  "{name: [J_arm_r_01, J_arm_r_02, J_arm_r_03, J_arm_r_04, J_arm_r_05, J_arm_r_06, J_arm_r_07],
    position: [0.5, -0.3, 0.0, 0.8, 0.0, 0.0, 0.0]}"
```

该入口不插值，仅建议用于调试。

### `/codroid/left_arm/pose_target`

### `/codroid/right_arm/pose_target`

- 类型：`geometry_msgs/msg/PoseStamped`
- 方向：轨迹节点订阅
- 坐标系：调用方必须提供 `base_link` 数据；当前不执行 TF 转换
- 姿态：ROS 四元数 `(x,y,z,w)`

```bash
ros2 topic pub --once /codroid/right_arm/pose_target \
  geometry_msgs/msg/PoseStamped \
  "{header: {frame_id: base_link}, pose: {
    position: {x: 0.30, y: -0.25, z: 0.20},
    orientation: {x: 0.0, y: 0.0, z: 0.0, w: 1.0}}}"
```

处理步骤：读取实际关节角 → IK → 限位检查 → 自动计算时长 → 三次插值 → 100 Hz 下发。

### `/codroid/arm_trajectory`

- 类型：`trajectory_msgs/msg/JointTrajectory`
- 方向：轨迹节点订阅

规则：

- 关节必须是完整单臂或完整双臂；
- 点的位置数量必须与关节名数量一致；
- 多点的 `time_from_start` 必须严格递增且为正；
- 单点允许时间为零，此时自动计算时长；
- 所有位置必须满足 URDF 限位。

```bash
ros2 topic pub --once /codroid/arm_trajectory \
  trajectory_msgs/msg/JointTrajectory \
  "{joint_names: [J_arm_r_01, J_arm_r_02, J_arm_r_03, J_arm_r_04, J_arm_r_05, J_arm_r_06, J_arm_r_07],
    points: [{positions: [0.5, -0.3, 0.0, 0.8, 0.0, 0.0, 0.0]}]}"
```

### `/codroid/trajectory_status`

- 类型：`std_msgs/msg/String`
- 状态：`active`、`completed`、`trajectory cancelled`、
  `no active trajectory` 或 `rejected: <原因>`

## 5. Service

| Service | 类型 | `true` | `false` |
|---|---|---|---|
| `/codroid_arm_bridge/connect` | `std_srvs/srv/SetBool` | UDP 连接 | 断开 |
| `/codroid_arm_bridge/enable` | `std_srvs/srv/SetBool` | 上使能 | 下使能 |
| `/codroid_arm_bridge/teleoperation` | `std_srvs/srv/SetBool` | 允许遥操作 | 停止遥操作 |
| `/codroid_arm_bridge/lock` | `std_srvs/srv/SetBool` | 获取控制权 | 释放控制权 |
| `/codroid_arm_bridge/left_control` | `std_srvs/srv/SetBool` | 开启左臂实时控制 | 停止 |
| `/codroid_arm_bridge/right_control` | `std_srvs/srv/SetBool` | 开启右臂实时控制 | 停止 |

错误复位：

```bash
ros2 service call /codroid_arm_bridge/reset_error std_srvs/srv/Trigger "{}"
```

取消当前插值：

```bash
ros2 service call /codroid_arm_trajectory/cancel std_srvs/srv/Trigger "{}"
```

取消插值不会自动发送 `StopControl`。

## 6. 推荐真机流程

确认反馈：

```bash
ros2 topic hz /joint_states
```

按现场安全流程进入控制：

```bash
ros2 service call /codroid_arm_bridge/enable std_srvs/srv/SetBool "{data: true}"
ros2 service call /codroid_arm_bridge/lock std_srvs/srv/SetBool "{data: true}"
ros2 service call /codroid_arm_bridge/right_control std_srvs/srv/SetBool "{data: true}"
```

停止顺序：

```bash
ros2 service call /codroid_arm_trajectory/cancel std_srvs/srv/Trigger "{}"
ros2 service call /codroid_arm_bridge/right_control std_srvs/srv/SetBool "{data: false}"
ros2 service call /codroid_arm_bridge/lock std_srvs/srv/SetBool "{data: false}"
ros2 service call /codroid_arm_bridge/enable std_srvs/srv/SetBool "{data: false}"
ros2 service call /codroid_arm_bridge/connect std_srvs/srv/SetBool "{data: false}"
```

## 7. IK 模式

### `controller`

通过 `ws://<robot_ip>:9000` 调用控制器 `action:"ik"`。ROS 四元数先转换为控制器要求的
`[x,y,z,roll,pitch,yaw]`。

### `local`

从 `arms_only.urdf` 构造运动链，使用当前实际关节角作为初值，采用阻尼最小二乘雅可比、
零空间关节居中、关节限位和单步增量限制。默认收敛阈值为位置 `0.2 mm`、姿态
`0.002 rad`。不收敛时拒绝目标。

### `controller_then_local`

先使用控制器 IK；连接失败、超时或返回无效时自动使用本地 IK。

## 8. 三次插值

```text
u = t/T
s(u) = 3u² - 2u³
q(t) = q0 + (q1-q0)s(u)
```

起止速度为零。单目标自动时长为：

```text
T = max(minimum_duration, 1.5 × max(|q1-q0|) / default_max_velocity)
```

## 9. RViz 纯显示测试

### 9.1 CoDroid Arm Control 面板

三个随包提供的 RViz 配置都会自动加载 `CoDroid Arm Control` 面板。面板包含：

- UDP 连接开关；
- 机器人使能开关；
- 控制权开关；
- 遥操作允许开关；
- 左右臂实时控制开关；
- 错误复位和轨迹取消；
- 左右臂选择以及 `base_link` 下的 XYZ、四元数目标输入。

使能、控制权、实时控制和 Pose 下发均有确认对话框。面板复用本文档第 4、5 节的
Topic/Service，不绕过桥接、IK、限位检查或插值节点。

### 9.2 纯显示挥臂

```bash
ros2 launch codroid_description wave_demo.launch.py
```

该入口直接发布模拟 `/joint_states`，不连接真机。不要与真机 launch 同时启动。

## 10. 安全限制

1. 首次上机应使用小位移和保守速度，并保证急停可触达；
2. 开启控制前确认实际反馈与 RViz 姿态一致；
3. 本地 IK 不进行自碰撞或环境碰撞检测；
4. `completed` 只表示命令插值完成，不代表已经验证真机跟踪误差；
5. 报警、网络异常或轨迹异常时，应停止实时控制并释放控制权。
